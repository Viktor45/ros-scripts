# 🔄 Скрипт DNS-резолвинга MikroTik

English version: [README.md](README.md) | Русская версия: [README_ru.md](README_ru.md)

Мощный и гибкий скрипт RouterOS, который автоматически резолвит IPv4 и IPv6 адреса в списках адресов фаервола и сохраняет разрешенные имена хостов в комментариях для удобной идентификации и отслеживания.

## ✨ Возможности

- **Поддержка двойного стека**: Полное разрешение адресов IPv4 и IPv6
- **Гибкий выбор версии IP**: Обработка только IPv4, только IPv6 или обоих одновременно
- **Автоматический DNS-резолвинг**: Автоматически резолвит IP-адреса в hostname
- **Настраиваемый DNS-сервер**: Используйте любой DNS-сервер по вашему выбору (Cloudflare, Google, OpenDNS и т.д.)
- **Раздельная конфигурация списков**: Независимая конфигурация для списков адресов IPv4 и IPv6
- **Обработка ошибок**: Встроенная обработка ошибок для неудачных DNS-резолвингов
- **Отладочное логирование**: Опциональное логирование для диагностики (закомментировано по умолчанию)
- **Простая конфигурация**: Все настройки в понятном разделе конфигурации
- **Недеструктивный**: Обновляет только пустые комментарии, сохраняя существующие данные
- **Надежная обработка**: Оптимизированный скрипт обрабатывает обе версии IP с проверенной логикой

## 🔧 Требования

- Роутер MikroTik с RouterOS v7.20 или выше
- Административный доступ к роутеру
- Сетевое подключение для DNS-резолвинга
- Подключение IPv6 (если обрабатываются адреса IPv6)

## 📥 Установка

1. Подключитесь к вашему роутеру MikroTik через Winbox, WebFig или SSH

2. Перейдите в **System → Scripts**

3. Нажмите кнопку **+** для создания нового скрипта

4. Дайте ему осмысленное имя (например, `resolve-address-lists`)

5. Скопируйте и вставьте код скрипта в поле source

6. Нажмите **OK** для сохранения

## ⚙️ Конфигурация

Отредактируйте раздел конфигурации в верхней части скрипта для настройки его поведения:

### Конфигурация DNS-сервера

```routeros
:local dnsServer "1.1.1.1"
```

**Популярные DNS-серверы:**
- `1.1.1.1` — Cloudflare DNS (по умолчанию, быстрый и ориентированный на конфиденциальность)
- `8.8.8.8` — Google DNS (надежный и широко используемый)
- `208.67.222.222` — OpenDNS (доступна семейная фильтрация)
- `9.9.9.9` — Quad9 (ориентированный на безопасность)
- `2606:4700:4700::1111` — Cloudflare DNS (IPv6)
- `2001:4860:4860::8888` — Google DNS (IPv6)
- DNS-сервер вашего провайдера

### Выбор версии IP

```routeros
:local ipVersion "both"
```

**Доступные опции:**
- `"ipv4"` — Обрабатывать только списки адресов IPv4
- `"ipv6"` — Обрабатывать только списки адресов IPv6
- `"both"` — Обрабатывать оба списка адресов IPv4 и IPv6 (по умолчанию)

### Конфигурация списков адресов IPv4

```routeros
:local ipv4ListsToResolve {
    "Trap";
    "MYDN";
    "MyCustomList";
}
```

Добавляйте или удаляйте имена списков адресов IPv4 по мере необходимости. Скрипт будет обрабатывать только записи из этих списков.

### Конфигурация списков адресов IPv6

```routeros
:local ipv6ListsToResolve {
    "Trap-v6";
    "MYDN-v6";
    "MyCustomList-v6";
}
```

Добавляйте или удаляйте имена списков адресов IPv6 по мере необходимости. Скрипт будет обрабатывать только записи из этих списков.

## 🚀 Использование

### Ручной запуск

Запустите скрипт из терминала:

```
/system script run resolve-address-lists
```

### Запланированное выполнение

Настройте автоматическое выполнение через регулярные интервалы:

1. Перейдите в **System → Scheduler**
2. Нажмите **+** для добавления нового расписания
3. Настройте:
   - **Name**: `auto-resolve-addresses`
   - **Start Date/Time**: Установите текущую дату/время
   - **Interval**: `1d 00:00:00` (запускается ежедневно)
   - **On Event**: `/system script run resolve-address-lists`

4. Нажмите **OK**

### Интеграция с другими скриптами

Вы можете вызывать этот скрипт из других скриптов:

```routeros
/system script run resolve-address-lists
```

## 🔍 Принцип работы

1. **Инициализация**: Загружает конфигурацию (DNS-сервер, версия IP и целевые списки адресов)

2. **Выбор версии**: Определяет, какие версии IP обрабатывать на основе настройки `ipVersion`

3. **Обработка IPv4** (если включено):
   - Проходит по всем записям списка адресов фаервола IPv4
   - Проверяет, совпадает ли имя списка записи с настроенными списками IPv4
   - Проверяет, пусто ли поле комментария
   - Пытается выполнить DNS-резолвинг с использованием настроенного DNS-сервера
   - Обновляет поле комментария разрешенным именем хоста или "error"

4. **Обработка IPv6** (если включено):
   - Проходит по всем записям списка адресов фаервола IPv6
   - Проверяет, совпадает ли имя списка записи с настроенными списками IPv6
   - Проверяет, пусто ли поле комментария
   - Пытается выполнить DNS-резолвинг с использованием настроенного DNS-сервера
   - Обновляет поле комментария разрешенным именем хоста или "error"

5. **Завершение**: Обе версии IP обрабатываются независимо на основе конфигурации

## 📖 Примеры

### Пример 1: Настройка только IPv4

Настройте скрипт для разрешения только в части адресов IPv4:

```routeros
:local dnsServer "8.8.8.8"
:local ipVersion "ipv4"
:local ipv4ListsToResolve {
    "Denylist";
    "Allowlist";
}
```

### Пример 2: Настройка только IPv6

Настройте скрипт для разрешения только в части адресов IPv6:

```routeros
:local dnsServer "2606:4700:4700::1111"
:local ipVersion "ipv6"
:local ipv6ListsToResolve {
    "Denylist-v6";
    "Allowlist-v6";
}
```

### Пример 3: Двойной стек (IPv4 + IPv6)

Одновременный мониторинг адресов IPv4 и IPv6:

```routeros
:local dnsServer "1.1.1.1"
:local ipVersion "both"

:local ipv4ListsToResolve {
    "Suspicious-IPs";
    "Failed-Logins";
    "Port-Scanners";
}

:local ipv6ListsToResolve {
    "Suspicious-IPs-v6";
    "Failed-Logins-v6";
    "Port-Scanners-v6";
}
```

### Пример 4: Двойной стек для нескольких сетей

Разрешение адресов в различных сегментах сети для обеих версий IP:

```routeros
:local dnsServer "208.67.222.222"
:local ipVersion "both"

:local ipv4ListsToResolve {
    "Guest-Network";
    "IoT-Devices";
    "Management-IPs";
    "VPN-Clients";
}

:local ipv6ListsToResolve {
    "Guest-Network-v6";
    "IoT-Devices-v6";
    "Management-IPs-v6";
    "VPN-Clients-v6";
}
```

### Пример 5: Мониторинг безопасности с раздельными DNS

Используйте разные DNS-серверы для IPv4 и IPv6 (требуется запуск скрипта дважды с разными конфигурациями):

**Первый запуск (IPv4):**
```routeros
:local dnsServer "8.8.8.8"
:local ipVersion "ipv4"
:local ipv4ListsToResolve { "Threats" }
```

**Второй запуск (IPv6):**
```routeros
:local dnsServer "2001:4860:4860::8888"
:local ipVersion "ipv6"
:local ipv6ListsToResolve { "Threats-v6" }
```

## 🐛 Диагностика

### Включение отладочного логирования

Раскомментируйте строки логирования в скрипте для просмотра детальной информации о выполнении:

```routeros
:log info ("*** Processing IPv4 addresses - DNS Server: " . $dnsServer);
:log info ("*** Processing IPv6 addresses - DNS Server: " . $dnsServer);
```

Просматривайте логи в меню **Log** или через терминал:

```
/log print where topics~"script"
```

### Распространенные проблемы

**Проблема**: Скрипт не обрабатывает никакие записи

**Решение**:
- Убедитесь, что имена ваших списков адресов точно совпадают с именами в `ipv4ListsToResolve` или `ipv6ListsToResolve`
- Проверьте, что `ipVersion` установлен правильно ("ipv4", "ipv6" или "both")
- Убедитесь, что у вас есть записи в соответствующих списках адресов (`/ip firewall address-list` или `/ipv6 firewall address-list`)

---

**Проблема**: Разрешение IPv6 показывает "error"

**Решение**:
- Проверьте, что ваш роутер имеет подключение IPv6
- Проверьте, что DNS-сервер поддерживает запросы IPv6
- Если используете DNS-сервер IPv6, убедитесь, что ваш роутер может достичь его
- Протестируйте подключение IPv6: `/ipv6 route print` и `/ipv6 address print`

---

**Проблема**: Все разрешения имен показывают "error"

**Решение**:
- Проверьте подключение к DNS-серверу
- Убедитесь, что IP-адрес DNS-сервера правильный
- Убедитесь, что правила фаервола разрешают DNS-запросы (UDP порт 53)
- Протестируйте DNS вручную: `/tool fetch url=http://www.google.com`

---

**Проблема**: Комментарии не обновляются

**Решение**: Скрипт обновляет только пустые комментарии. Для принудительного повторного разрешения сначала очистите существующие комментарии:

```routeros
# Очистить комментарии IPv4
/ip firewall address-list set [find list="YourListName"] comment=""

# Очистить комментарии IPv6
/ipv6 firewall address-list set [find list="YourListName-v6"] comment=""
```

---

**Проблема**: Скрипт выполняется слишком долго

**Решение**:
- Уменьшите количество списков в `ipv4ListsToResolve` и `ipv6ListsToResolve`
- Установите `ipVersion` в "ipv4" или "ipv6" вместо "both", если двойной стек не нужен
- Уменьшите количество записей в ваших списках адресов
- Разделите обработку на отдельные запланированные запуски для IPv4 и IPv6

---

**Проблема**: Разные результаты для IPv4 и IPv6

**Решение**: Это нормальное поведение. Некоторые хосты могут иметь только IPv4 (A-записи) или IPv6 (AAAA-записи), или могут разрешаться в разные hostname в зависимости от версии IP.

## 🌐 Лучшие практики IPv6

### Соглашение об именовании
Используйте согласованное именование для списков IPv4 и IPv6:
- IPv4: `Denylist`, `Allowlist`, `Suspicious-IPs`
- IPv6: `Denylist-v6`, `Allowlist-v6`, `Suspicious-IPs-v6`

### Выбор DNS-сервера
- Используйте DNS-серверы IPv4 для общего использования (работает для разрешения адресов как IPv4, так и IPv6)
- Используйте DNS-серверы IPv6 только если у вас стабильное подключение IPv6
- Популярные DNS-провайдеры хорошо работают с обеими версиями IP

### Организация списков адресов
Разделяйте списки адресов IPv4 и IPv6 для:
- Более простого управления и диагностики
- Независимого контроля обработки
- Четкой видимости в правилах фаервола
