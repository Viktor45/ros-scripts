## 🌟 Обзор

Этот скрипт RouterOS автоматизирует процесс поиска и настройки рабочих эндпоинтов WireGuard для подключений Cloudflare WARP на роутерах MikroTik. Он интеллектуально тестирует различные комбинации IP/порт из инфраструктуры Cloudflare для установления наиболее надежного соединения.

### Зачем этот скрипт?

- **Автоматический отказоустойчивость**: Не требуется ручная настройка эндпоинта
- **Оптимальная производительность**: Автоматическое тестирование и выбор рабочих эндпоинтов
- **Устойчивость сети**: Быстрое восстановление после сбоев эндпоинта
- **Экономия времени**: Устраняет ручную настройку методом проб и ошибок
- **Готовность к продакшену**: Комплексная обработка ошибок и логирование

## 📦 Требования

### Оборудование
- **Роутер MikroTik** с поддержкой WireGuard
  - Любое устройство, совместимое с RouterOS v7.x
  - Достаточные CPU/RAM для шифрования WireGuard

### Программное обеспечение
- **Версия RouterOS**: 7.20 или выше
- **Пакет WireGuard**: Включен и настроен
- **Активное подключение к интернету**: Для тестирования эндпоинтов

### Предварительные условия
- Аккаунт Cloudflare WARP и учетные данные
- Интерфейс WireGuard уже создан и настроен
- Базовое понимание MikroTik RouterOS

## ⚙️ Конфигурация

### Базовая конфигурация

Отредактируйте следующие переменные в скрипте перед запуском:

```routeros
# Имя интерфейса WireGuard (должно совпадать с существующим интерфейсом)
:local wgInterface "cloudflare-interface"

# Адрес для ping для проверки подключения
:local checkAddress "1.1.1.1"

# Максимальное количество попыток эндпоинта
:local maxAttempts 10

# Количество ping-пакетов для отправки за тест
:local pingCount 2

# Задержка в секундах после изменения эндпоинта
:local delayTime 2

# Включить отладочное логирование (0=отключено, 1=включено)
:global DEBUG 1
```

### Определение имени интерфейса WireGuard

```routeros
# Список всех интерфейсов WireGuard
/interface wireguard print

# Пример вывода:
# 0  name="wgcf" mtu=1420 listen-port=51820
```

Используйте значение `name` для переменной `wgInterface`.

## 🔍 Как это работает

### Диаграмма рабочего процесса

```
┌─────────────────────────────────────────────────────────────┐
│  1. Инициализация скрипта и загрузка конфигурации           │
└─────────────────┬───────────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────────┐
│  2. Валидация массивов и создание временного маршрута       │
└─────────────────┬───────────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────────┐
│  3. Генерация случайного эндпоинта (IP:Port)                │
│     • Случайный префикс IP из списка                        │
│     • Случайный последний октет (1-254)                     │
│     • Случайный порт из списка                              │
└─────────────────┬───────────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────────┐
│  4. Настройка пира WireGuard                                │
│     • Отключить пир                                         │
│     • Обновить порт прослушивания                           │
│     • Установить адрес и порт эндпоинта                     │
│     • Повторно включить пир                                 │
└─────────────────┬───────────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────────────────────────────┐
│  5. Проверка подключения (Ping-тест)                        │
└─────────────────┬───────────────────────────────────────────┘
                  │
          ┌───────┴────────┐
          │                │
          ▼                ▼
    ┌─────────┐      ┌─────────┐
    │ Успех   │      │ Неудача │
    └────┬────┘      └────┬────┘
         │                │
         │                └──► Повтор (макс. попыток)
         │
         ▼
┌─────────────────────────────────────────────────────────────┐
│  6. Очистка и отчет                                         │
│     • Удалить временный маршрут                             │
│     • Логировать финальный статус                           │
└─────────────────────────────────────────────────────────────┘
```

### Пошаговый процесс

1. **Инициализация**
   - Загружает списки префиксов IP и портов
   - Валидирует конфигурацию
   - Создает запись временной маршрутизации

2. **Случайная генерация**
   - Использует системное время и uptime для энтропии
   - Генерирует случайный IP из адресного пространства Cloudflare
   - Выбирает случайный порт из известных рабочих портов

3. **Настройка**
   - Временно отключает пир WireGuard
   - Обновляет настройки эндпоинта
   - Изменяет порт прослушивания для избежания конфликтов
   - Повторно включает пир

4. **Тестирование**
   - Ждет handshake WireGuard
   - Отправляет ping-пакеты через туннель
   - Проверяет подключение

5. **Решение**
   - Если успешно: Выход и логирование успеха
   - Если неудачно: Пробует следующий случайный эндпоинт
   - Повторяет до успеха или достижения макс. попыток

6. **Очистка**
   - Удаляет временные маршруты
   - Предоставляет детальный отчет о статусе

## 🐛 Диагностика

### Распространенные проблемы

#### 1. "No WireGuard peer found"

**Причина**: Имя интерфейса WireGuard не совпадает или пир не настроен

**Решение**:
```routeros
# Проверить конфигурацию WireGuard
/interface wireguard print
/interface wireguard peers print

# Обновить скрипт правильным именем интерфейса
:local wgInterface "your-interface-name"
```

#### 2. "IP prefixes array is empty"

**Причина**: Инициализация массива не удалась

**Решение**:
```routeros
# Вручную проверить синтаксис массива
:global ipPrefixes [:toarray "8.6.112.,162.159.192."]
:put [:len $ipPrefixes]
# Должно вывести: 2
```

#### 3. Рабочий эндпоинт не найден

**Причина**: Проблемы сети, блокировка фаерволом или устаревшие эндпоинты

**Решение**:
```routeros
# Проверить правила фаервола
/ip firewall filter print

# Убедиться, что UDP разрешен
/ip firewall filter add chain=output action=accept protocol=udp

# Попробовать увеличить maxAttempts
:local maxAttempts 50

# Обновить списки IP/портов текущими эндпоинтами Cloudflare
```

#### 4. Скрипт запускается, но соединения нет

**Причина**: Настройка маршрутизации или фаервола

**Решение**:
```routeros
# Проверить, что интерфейс WireGuard активен
/interface print stats

# Проверить таблицу маршрутизации
/ip route print

# Протестировать ручной ping
/ping 1.1.1.1 interface=wgcf
```

### Режим отладки

Включить детальное логирование:

```routeros
:global DEBUG 1
```

Проверить логи:

```routeros
# Просмотреть недавние логи
/log print where topics~"script"

# Фильтровать логи предупреждений
/log print where message~"DEBUG"
```

### Ручное тестирование

Протестировать эндпоинт вручную:

```routeros
# Установить конкретный эндпоинт
/interface wireguard peers set [find interface=wgcf] \
  endpoint-address=162.159.192.1 \
  endpoint-port=500

# Протестировать подключение
/ping 1.1.1.1 count=10 interface=wgcf
```
